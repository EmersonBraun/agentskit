`useChat` gains `edit` and `regenerate`.

### `regenerate(messageId?)`

Regenerate an assistant response. With no id, regenerates the last turn (same shape as `retry`). With an id, targets that specific assistant message and drops every turn after it.

```ts
const chat = useChat({ adapter })
// Hover a previous assistant message and offer a "regenerate" button:
await chat.regenerate(assistantMessage.id)
```

### `edit(messageId, newContent, opts?)`

- **Assistant message** — content is replaced in place. No regeneration. Good for UIs that let reviewers correct a model's answer inline.
- **User message** — all subsequent turns are dropped, and the assistant reply is regenerated by default. Pass `{ regenerate: false }` to just truncate.

```ts
// Edit the model's answer in place
await chat.edit(assistantMessage.id, 'Corrected: ...')

// Edit a user prompt and re-run
await chat.edit(userMessage.id, 'actually, can you use Python instead?')

// Truncate without re-running
await chat.edit(userMessage.id, 'rephrased', { regenerate: false })
```

Optimistic-UI contract: the state is updated before the network round-trip so your UI re-renders immediately.

Both methods are exposed on `ChatReturn` (React + Ink hooks) and `ChatController` (headless).
