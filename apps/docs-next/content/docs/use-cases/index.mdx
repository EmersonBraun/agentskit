If packages tell you what AgentsKit contains, use cases tell you what it helps you ship.

These guides are opinionated starting points. Each one maps a real product outcome to the packages, recipes, and production layers you will likely combine.

## Pick the agent you want to build

- [Support agent](./support-agent) — customer support, ticket triage, tool calls, escalation paths.
- [Research agent](./research-agent) — web research, source collection, summaries, background jobs.
- [Code agent](./code-agent) — repository analysis, code review, file tools, multi-step reasoning.
- [Internal copilot](./internal-copilot) — company knowledge, RAG, permissions, observability, approvals.

## How to use this section

1. Start with the use case closest to your product.
2. Copy the suggested package set and architecture.
3. Follow the linked recipes and production guides to harden it.
